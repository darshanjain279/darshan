DP on Grids

Not started yet — 7 problems queued.

2D DP where dp[i][j] depends on dp[i-1][j] and dp[i][j-1] (or similar neighbors), often with an obstacle or cost twist.

Sub-patterns

  • A — Path Counting. How many distinct paths exist from one corner to another.
  • B — Path Cost Minimization. Cheapest path from one corner to another.
  • C — Obstacle Handling. Blocked cells that break the simple recurrence.
  • D — Grid DP with Extra Carried State. State includes more than just position, e.g. remaining moves or collected items.
